# Acesso de Emergência — Pedal Data

## Cenários
- Ausência inesperada do fundador
- Perda de acesso a contas críticas
- Indisponibilidade de serviços essenciais

## Itens protegidos
| Item | Proteção | Acesso de emergência |
|---|---|---|
| Gestor de senhas | MFA + recovery codes | Cofre físico + pessoa designada |
| Domínio | Registrar lock | Acesso compartilhado no registrar |
| GitHub | MFA + recovery codes | Pessoa designada como owner |
| Servidores | Chave SSH + VPN | Chave de backup em cofre |
| Backups | Criptografia + offsite | Chave de descriptografia em cofre |
| Conta do provedor de email | MFA + recovery | Pessoa designada |

## Procedimento
1. Pessoa designada acessa cofres físicos com instruções
2. Recupera senhas do gestor de senhas
3. Acessa repositórios e servidores
4. Mantém operação mínima (site no ar, dados acessíveis)
5. Transfere domínios e contas definitivamente em até 30 dias
